Focus only on how PhishGuard displays detection results to users.

Evaluate and improve the result card/section based on these security tool UX principles:

1. VERDICT HIERARCHY: Is the verdict (Malicious/Benign) the most visually dominant element?
2. CONFIDENCE COMMUNICATION: Does the UI make 91% vs 67% confidence feel meaningfully different to users?
3. REASON READABILITY: Is the reason text written in plain English a non-technical user can understand?
4. THREAT FLAGS: Are individual threat flags (typosquatting, suspicious TLD, etc.) shown in a scannable way?
5. FALSE POSITIVE TRUST: Is there any affordance for users to report a wrong verdict?
6. COPY/SHARE: Can users easily copy the result or share a permalink?

Suggest and implement specific improvements. Show before/after code for any changes.
